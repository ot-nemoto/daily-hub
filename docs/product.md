# product.md — プロダクト定義

## プロダクト名

daily-hub

## 目的

チームメンバーが毎日の業務内容・翌日予定・所感を記録・共有する日報管理ツール。
個人の振り返り習慣化と、チーム間の透明性向上を両立する。

## 対象ユーザー

- エンジニア・デザイナー・PM などの少人数〜中規模チーム
- 技術的知識がなくても使えるWebブラウザベースのUI

## 解決する課題

| 課題 | daily-hub での解決 |
|------|-------------------|
| 日報が各自のメモ・チャットに散在する | 一元管理されたWebアプリで記録・閲覧 |
| 他のメンバーの状況が把握しにくい | 日次ビューで全員の日報をまとめて閲覧 |
| 過去の振り返りが困難 | 月次ビュー・期間選択で自分の日報を一覧 |
| フィードバックがしにくい | コメント機能でその日の日報に直接返信 |

## MVPスコープ（現フェーズ）

- 認証（メール/パスワード）
- 日報の作成・編集（作業内容 / 明日の予定 / 感想・課題・問題点）
- 他ユーザーの日報閲覧
- コメント追加
- 日次ビュー（日付選択 + ユーザー選択）
- 月次ビュー（期間選択）

## フェーズ 7 スコープ（ユーザー管理）

### Phase 7a — 最小構成（管理者・ロール・無効化）
- ロール管理（`admin` / `member`）
- 管理画面へのアクセス制限（`admin` ロールのみ）
- ユーザー一覧表示（名前・メール・ロール・登録日・最終日報投稿日・直近30日提出率）
- ロール変更・アカウント無効化（データを保持したままログイン不可）

### Phase 7b — ユーザー招待・追加
- 管理者によるユーザー直接作成（名前・メール・仮パスワード）
- 招待リンク発行（期限付きトークン、招待先メール任意指定）

### Phase 7c — 詳細管理
- `viewer` ロール追加（閲覧・コメントのみ、日報作成不可）
- パスワードリセット（管理者が仮パスワードを発行）
- アカウント削除（日報・コメントを含む完全削除）

## MVPスコープ外（将来フェーズ）

- Slack/Email リマインダー通知
- 承認ワークフロー
- CSV/PDFエクスポート
- 組織/チーム単位のマルチテナント
- 全文検索

## 成功指標（MVP）

- チームメンバー全員が週5日継続して日報を提出できること
- 他メンバーの日報へのコメントが週1件以上発生すること
