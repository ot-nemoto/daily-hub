# E2Eテストシナリオ・手動テスト観点

各機能を実装・修正したときに確認すべき観点をまとめたもの。
「この機能を触れば、ここが確認できる」という指標として使う。

---

## 認証・リダイレクト

未ログイン状態で保護ルートにアクセスしたとき `/login` にリダイレクトされるかを確認する。

- `/reports/daily` / `/reports/monthly` / `/reports/new` / `/reports/[id]` / `/settings` / `/admin/users` に未ログインでアクセス
- サインイン成功後に `/reports/daily` に戻るか
- `isActive=false` のユーザーでログインしたとき `/auth-error?reason=inactive` にリダイレクトされ「アカウントが無効化されています」と表示されるか
- `/auth-error` でサインアウトボタンを押すと `/login` に遷移するか
- ヘッダーのログアウトボタンを押すと `/login` に遷移するか

---

## ヘッダー

- ヘッダーに「日次ビュー」「月次ビュー」「日報作成」「ユーザー名」「ログアウト」が表示されるか
- ADMIN ロールのみ「ユーザー管理」リンクが表示されるか
- VIEWER ロールでは「日報作成」リンクが非表示か

## 日報一覧（デイリービュー）

- 今日の日付が初期表示されるか
- 自分の日報が表示されるか
- 0件のとき「YYYY年M月D日 の日報はありません」と表示されるか（例: 「2026年4月4日 の日報はありません」）
- 他ユーザーの日報も一覧に表示されるか
- ユーザーフィルターで特定ユーザーの日報のみ絞り込めるか
- 不正な日付（不完全・存在しない日付）を入力したとき赤枠表示され、URLが更新されないか
- 有効な日付を入力したとき即時反映されるか
- 日報カードに「詳細」ボタンが表示され、日報詳細に遷移するか
- 自分の日報カードに「編集」ボタンが表示されるか
- 他ユーザーの日報カードに「編集」ボタンが表示されないか

## 日報一覧（マンスリービュー）

- デフォルトで自分の日報が今月分表示されるか
- 0件のとき「YYYY年M月 の日報はありません」と表示されるか（例: 「2026年4月 の日報はありません」）
- ユーザーフィルターで他のユーザーの日報を切り替えられるか
- 不正な月（不完全・存在しない値）を入力したとき赤枠表示され、URLが更新されないか
- 有効な月を入力したとき即時反映されるか

## 日報作成

- 日付フィールドのデフォルトが今日の日付か
- 業務内容・明日の予定・所感を入力して保存するとデイリービューに反映されるか
- 業務内容が空のとき保存できないか（バリデーション）
- 明日の予定が空のとき保存できないか（バリデーション）
- 同じ日付で2件目を作成しようとしたとき 409 エラーが表示されるか
- 保存後に日報詳細ページに遷移するか
- 保存中に送信ボタンが「保存中...」になり無効化されるか
- VIEWER ロールで `/reports/new` にアクセスすると `/reports/daily` にリダイレクトされるか
- キャンセルボタンでデイリービューに戻るか

## 日報詳細

- 業務内容・明日の予定・所感が正しく表示されるか
- 自分の日報に編集ボタンが表示されるか
- 他ユーザーの日報に編集ボタンが表示されないか
- コメントが0件のとき「コメントはありません」と表示されるか

### APIキー管理

- APIキー未生成時に「生成する」ボタンが表示されるか
- 「生成する」をクリックするとキーが生成され、プレーンテキストで表示されるか
- 「隠す」をクリックするとマスク表示に切り替わるか
- 「表示」をクリックするとプレーンテキスト表示に切り替わるか
- ページリロード後にキーがマスク表示で保持されているか
- 「再生成」をクリックすると新しいキーに更新されるか
- 「失効」をクリックするとキーが削除され「生成する」ボタンに戻るか
- 別ユーザーのキーが表示されないか（ユーザー分離）

## REST API（外部連携）

以下は curl などの外部クライアントで手動確認する。

- 有効な API キーで `POST /api/reports` を呼ぶと 201 と `{ id }` が返るか
  ```bash
  curl -X POST http://localhost:3000/api/reports \
    -H "Authorization: Bearer <api-key>" \
    -H "Content-Type: application/json" \
    -d '{"date":"YYYY-MM-DD","workContent":"作業内容","tomorrowPlan":"明日の予定","notes":""}'
  ```
- Authorization ヘッダーなしで 401 が返るか
- 無効な API キーで 401 が返るか
- VIEWER ロールのユーザーのキーで 403 が返るか
- 同日に日報が既存の場合 409 が返るか
- `date` が `YYYY-MM-DD` 形式でない場合 422 が返るか
- 日報一覧（日次ビュー）に API 経由で作成した日報が表示されるか

## 管理画面（ADMINのみ）

- MEMBER / VIEWER ロールでは `/admin/users` にアクセスできないか
- ユーザー一覧に「名前・メール・ロール・登録日・最終日報投稿日・直近30日提出率」が表示されるか
- ユーザーのロールを変更できるか
- ユーザーを無効化（isActive=false）すると該当ユーザーが `/auth-error?reason=inactive` にリダイレクトされるか
- 無効化済みアカウントを再有効化できるか
- 自分自身のロール降格・有効化/無効化ができないか

## ユーザー分離

- 別ユーザーでログインしたとき他ユーザーの日報編集 URL に直アクセスすると詳細ページにリダイレクトされるか
- 自分の日報のみ編集ボタンが表示されるか
- 自分のコメントのみ削除ボタンが表示されるか
